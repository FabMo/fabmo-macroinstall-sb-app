Information About Handibot/FabMo MACROS

Marcros are small, simple pieces of code that help you carry out useful tasks. They can be called 'scripts' or 'canned cycles' or even 'shortcuts'. At ShopBot, we have been calling them 'custom cuts' ... but 'MACRO' sounds less complex and more friendly so we'll be using that term in FabMo. Macros tend to be things you want to do over-and-over again so it is nice to be able to stash them away in a convenient place. Macros are found in the FabMo Macro Manager that you acceess from the left-side panel. There you can run, view, edit, and create Macros.

What's in a Macro?

FabMo will eventually support the writing of Macros in a number of languages, but at the moment, Macros are created in the OpenSBP digital-fab language (the language used by ShopBots). We use OpenSBP simply because it is easily understood and readily programmed to generate flexible routines. You can explore OpenSBP in the SB4 App, where there is also documentation in the HELP menu. Or, you can explore a Macro like Macro#3 that will give you some pretty good tutoring on how to roll-your-own.

What's this particular App All About?

What this app does is install the Macros that are appropriate for your tool, or allow you to install 1 or more if you lose them or they get corrupted. You won't need an internet connection for the install, if you have the app on your tool. This Macro Install App, and others, are available from docs.handibot.com, so feel free to delete this app after you have made use it. You can do that from the FabMo Configuration Manager found on the left-side panel, Apps section.

Macro Numbering Conventions

In therory, you can number your Macros any way that you would like. However, ShopBot observes a couple of conventions. Macros# 1-9 are used for a set of standard functions that are called by Job Files created in V-Carve and Fusions 360. Similarly, Macros# 90-99 are used for ShopBot/Handibot functions. So working between #10 and #89 is best for your own Macros. In particular, we use Macro#3 for zeroing and Macros #6 and #7 for spindle control.

Macros can be called from within Job Files that are written in OpenSBP. This allows them to function like subroutines or canned cycles. You will notice that files posted from a V-Carve Pro, for example will call Macros as a standard method for turning on and off the router/spindle.

To make a call from your code, use the expression: C#,xx (where the Macro number is placed after the comma -- FYI, this is the format for ShopBot's CustomCut call). Using this convention, you can also call Macros from within Macros, or you can call a Macro from an App that you create.
